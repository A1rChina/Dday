# 开发与重构记录：订单管理模块“客户-项目-产品”精密联动

## 1. 初始背景与问题描述
**核心诉求**：在新增/编辑订单时，要求实现“客户”、“关联项目”与“产品明细”的三向精密联动。
**原始问题**：
1. **数据失联**：系统引入了 Refine 框架，其底层的 `useSelect` 钩子会在异步加载字典时产生 ID 与文本名称的匹配竞态条件，导致虽然选择了客户，但实际上底层保存的是 `customer_id`（如 `"cust_changying"`），从而无法正确过滤出 `customer_name === "长盈"` 的项目和产品。
2. **交互越权**：之前的“智能”联动逻辑“用力过猛”——点选客户后会强行自动填入名下的第一个项目并瞬间铺满明细。如果客户有多个项目，这种强制预填极大地干扰了用户的操作直觉。

## 2. 架构方案与思路重构
为了彻底解决此问题，我们采取了“解耦框架、回归直觉”的重构思路：

### 2.1 抛弃低代码黑盒，采用底层状态驱动
弃用带黑盒特性的 `useSelect`，改用纯粹的 `useList` 主动抓取 `customers` 和 `products` 的全量/分页数据。在前端将下拉选择器的 `value` 严格绑定为真实的**纯文本名称**（`c.name`），彻底消除一切在前端进行 ID $\leftrightarrow$ Name 互转的 Hack 补丁，确保发往后端的 payload 是极致纯净的。

### 2.2 确立交互规范（方案 A：标准直觉级联）
我们通过讨论，确定了最符合业务直觉、最能“减少人工输入”但不抢夺用户控制权的交互路径：
- **第一级（选客户）**：仅负责**过滤**出对应的项目选项，并**静默清空**当前项目和网格。绝不自作主张强制填入项目。
- **第二级（选项目）**：一旦用户亲手点选了某个项目，系统瞬间在下方的网格中**自动铺满**该项目名下的所有精密型材产品（带默认数量和交期）。
- **第三级（逆向反推）**：如果用户直接在明细网格中新增了一个产品，系统自动往上反推并填入对应的客户和项目。

## 3. 踩坑记录与极致排查 (关于 "No Data" 事件)
在实施重构的过程中，遇到了一次极其惊险的“下拉框报 No Data (暂无数据)”故障。排查全过程如下：

1. **第一次排查（数据解构层）**：
   发现 Refine v5 版本的 `useList` 配合 React Query 时，其返回的响应体会包裹两层 `data`。即 `customerData.data.data` 才是真正的数组。由于我们最初使用 `(customerData?.data || [])` 去 `.map()`，导致解析失败。我们迅速修正了结构 `((customerData?.data?.data || customerData?.data) as any[] || [])`。
   *结果*：依旧 "No Data"。

2. **第二次排查（物理数据库验证）**：
   怀疑是否远端数据库中真的没有数据？我们直接通过终端调用 `npx wrangler d1 execute taskflow --remote --command="SELECT * FROM customers"`，成功查到了 2 条“长盈”的真实数据记录，并同时查到 `products` 表中有 8 条关联记录。
   *结论*：数据库完全正常，云端数据完好。

3. **第三次排查（致命的根源发现）**：
   进一步审查前端代码流转，终于发现了隐蔽的“幕后黑手”：**资源未注册**。
   在重构过程中，我们在 `OrderResource.tsx` 里调用了 `useList({ resource: 'customers' })`，但是在前端全局最高层的入口文件 `src/admin/main.tsx` 中，`<Refine resources={[...]}>` 数组里**竟然唯独遗漏了注册 `customers` 这个节点！**
   在 Refine 的严苛机制下，一旦查询的资源未在全局注册，底层拦截器就会直接静默抛弃该 HTTP 强求，导致组件根本没有向后端发起网络请求，从而返回空数据 "No Data"。

## 4. 最终修复与交付
1. 在 `main.tsx` 补上了缺失的 `{ name: 'customers', list: '/customers' }`。
2. 重新执行 `npm run typecheck` 和 `npm run build`，编译零警告。
3. 执行 `npm run deploy` 成功热推到 Cloudflare 生产环境。
4. **最终效果**：客户、项目、产品三向级联完美符合预期，不仅彻底拔除了强制预填的干扰，且底层数据绑定异常坚固。
